# 競品分析與功能建議

參考 NoWaste / FoodShiner / KitchenPal / Fridge Hero 等頂尖冰箱管理 App。

---

| 功能 | 頂尖 App | 本專案 | 落差 |

## 推薦新增功能（依可行性排列）

### 🥇 第一梯次：高價值、低工程成本

#### 1. 購物清單引擎
```
  概念：缺貨自動提醒 + 手動加入採買清單
  流程：
    食材歸零 → 自動加入購物清單
    手動新增 → 加入購物清單
    採買完成 → 掃入庫存 (未來)
  檔案：js/shopping-list.js (新建)
  儲存：shopping-list (LocalStorage key)
```

#### 2. 自訂類別管理
```
  概念：擴充 constants.js 的固定 6 類別
  流程：
    設定頁 → 新增/刪除類別 + 自訂天數
    e.g., "零食 14 天"、"麵包 3 天"
  檔案：js/storage.js (新增 getCategories/addCategory)
  儲存：fridge-categories (LocalStorage key)
```

#### 3. 食品浪費金額追蹤
```
  概念：每項食材記錄購買價格，完食後計算浪費金額
  流程：
    新增食材 → 可選填「購買價格」
    metric.js → 計算「總浪費金額 = Σ(價格×浪費%)」
    儀表板顯示「本月節省 NT$XXX」
  檔案：js/model.js (+price), js/metric.js (改)
```

#### 4. 資料匯出 / 匯入
```
  概念：備份全部資料為 JSON 檔案
  流程：
    設定頁 →「匯出備份」→ 下載 .json
    設定頁 →「匯入備份」→ 選擇檔案 → 覆蓋還原
  檔案：js/backup.js (新建)
```

#### 5. 批次操作模式
```
  概念：長按進入選擇模式，可批次標記完食或刪除
  流程：
    長按卡片 → 進入多選模式 (checkbox 浮現)
    勾選多項 → 底部工具列「批次完食」「批次刪除」
  檔案：js/canvas-ui.js (改)
```

---

### 🥈 第二梯次：中價值、中工程成本

#### 6. 條碼掃描 (Barcode Scanner)
```
  概念：使用 Barcode Detection API 或 ZXing.js
  流程：
    FAB 旁新增「掃描」按鈕
    → 開啟相機 → 掃描條碼
    → 查詢 Open Food Facts API (離線 fallback)
    → 自動填入名稱、類別、常見保存天數
  檔案：js/barcode.js (新建)
  依賴：ZXing.js (CDN) 或 BarcodeDetector API
```

#### 7. 庫存為基礎的食譜建議
```
  概念：根據快過期食材，推薦可做的料理
  流程：
    快過期食材清單 → 匹配預設食譜庫
    → 顯示「可用這些食材做：歐姆蛋、煎餅…」
  檔案：js/recipe.js (新建), data/recipes.json (新建)
```

#### 8. 快速新增模板
```
  概念：常用食材一鍵新增（常購清單）
  流程：
    設定常用食材 + 預設類別/單位/位置
    Modal 內顯示「常用」分頁 → 點擊即新增
  檔案：js/storage.js (新增 template 函數)
```

#### 9. 智慧到期日解析
```
  概念：支援輸入「6/25」或「3天後」自動轉為日期
  流程：
    到期日輸入框 → 解析自然語言
    → "明天" → 今日+1、"下週五" → 計算
  檔案：js/main.js (改 initModal)
```

### 🥉 第三梯次：高價值、高工程成本（未來規劃）

| 功能 | 說明 | 關鍵挑戰 |
|------|------|---------|
| 家庭共享 (Family Sync) | 多人共用同一冰箱，即時同步 | 需要後端/WebSocket/Firebase |
| 語音輸入 (Voice Input) | "Hey Siri, add milk" | Web Speech API，iOS 限制多 |
| Apple Watch 配件 | 手腕上查看快過期清單 | 需獨立 watchOS app |
| 照片辨識 (Photo Recognition) | 拍發票 → 自動辨識食材 | 需 OCR + AI，離線困難 |
| 營養追蹤 (Nutrition) | 計算每日攝入熱量/蛋白質 | 需龐大營養資料庫 |
| 智慧購物建議 | "牛奶快沒了，要加入購物清單嗎？" | 需 ML 預測模型 |

## 建議執行順序

```
本次 Task → 1.購物清單 → 2.自訂類別 → 3.金額追蹤
           → 4.資料匯出匯入 → 5.批次操作
           → 6.條碼掃描 → 7.食譜建議
           → 8.快速模板 → 9.智慧日期解析
           (→ 家庭共享 / 語音 / Watch → 未來)
```
